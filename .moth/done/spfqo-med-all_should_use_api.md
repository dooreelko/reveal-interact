update plugin and web lib to use @arinoto/cdk-arch@0.4.0 and api from architecture using strongly typed createHttpBindings

## Specification

### Goal
Replace manual HTTP request construction in plugin and web lib with strongly-typed API clients generated by `createHttpBindings` from `@arinoto/cdk-arch@0.4.0`.

### Overview
The `createHttpBindings` function generates typed HTTP client functions from an `ApiContainer`'s route definitions. This provides:
- Type safety: Function signatures match the architecture's handler types
- DRY: Route paths defined once in architecture, not duplicated in clients
- Consistency: All clients use the same route definitions

### API Routes Used

**Plugin (Host client):**
- `newSession`: `POST /api/v1/session/new` - Creates session, receives sessionUid/hostUid
- `setState`: `POST /api/v1/session/{sessionUid}/state/{page}/{state}` - Updates session state

**Web Library (User client):**
- `getSession`: `GET /api/v1/session/{sessionUid}` - Public, fetches session info
- `login`: `POST /api/v1/session/{sessionUid}/login` - User login
- `react`: `POST /api/v1/session/{sessionUid}/user/{uid}/react/{page}/{reaction}` - Send reaction
- `getState`: `GET /api/v1/session/{sessionUid}/state` - Get current state

### Authentication Pattern

Both clients need to send authentication headers. `createHttpBindings` accepts a custom `Fetcher` parameter that can add headers:

```typescript
const authFetcher = (token: string) => () => ({
  fetch: (url: string, options: RequestInit = {}) =>
    fetch(url, {
      ...options,
      credentials: "include",
      headers: {
        ...options.headers,
        "x-session-token": token,
      },
    }),
});

const client = createHttpBindings(endpoint, api, ['login', 'react'], authFetcher(token));
```

### Browser Crypto Compatibility

The `@revint/arch` package imports Node.js `crypto` module at the top level for server-side token verification. For browser compatibility:

**Plugin**: Uses esbuild with `@esbuild-plugins/node-modules-polyfill` to polyfill Node.js crypto module for browser environments.

**Web Library**: Same approach - esbuild with Node.js polyfills.

Note: The crypto functions are never called in browser context (only the route paths are used), but the module import still needs to resolve.

## Decisions

### Package Dependencies
- **Accepted:** Plugin and web lib add dependencies on `@revint/arch` and `@arinoto/cdk-arch`
- **Reason:** Required to use `createHttpBindings` with type safety from architecture

### Crypto Polyfill
- **Accepted:** Use `@esbuild-plugins/node-modules-polyfill` for browser builds
- **Reason:** Matches existing patterns (Vite for web example, similar to Cloudflare external approach)
- **Rejected:** Restructuring arch package to lazy-import crypto - more invasive change

### Custom Fetcher for Auth
- **Accepted:** Create auth wrapper fetcher that adds credentials and token header
- **Reason:** `createHttpBindings` default fetcher only sets Content-Type, auth headers needed

### Route Selection
- **Accepted:** Each client only binds the routes it needs
- **Plugin binds:** `newSession`, `setState`
- **Web lib binds:** `getSession`, `login`, `react`, `getState`
- **Reason:** Minimizes API surface, clear separation of host vs user operations

## Implementation Details

### Plugin Changes
1. Add dependencies: `@revint/arch`, `@arinoto/cdk-arch`, `@esbuild-plugins/node-modules-polyfill`
2. Replace `createSession()` with `createHttpBindings` for `newSession`
3. Replace `setState()` with `createHttpBindings` for `setState`
4. Update build scripts to use polyfill plugin
5. Custom fetcher handles auth headers and credentials

### Web Library Changes
1. Add dependencies: `@revint/arch`, `@arinoto/cdk-arch`, `@esbuild-plugins/node-modules-polyfill`
2. Replace manual fetch calls with `createHttpBindings` client
3. Update build script to use polyfill plugin
4. Custom fetcher handles auth headers and credentials

### Fetcher Pattern
```typescript
type AuthFetcher = (token: string) => Fetcher;

const createAuthFetcher: AuthFetcher = (token) => () => ({
  fetch: async (url, init = {}) =>
    globalThis.fetch(url, {
      ...init,
      credentials: "include",
      headers: {
        ...init.headers,
        "x-session-token": token,
      },
    }),
});
```
